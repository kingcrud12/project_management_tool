Architecture & Guidelines Frontend

Guide technique complet pour le développement du client Web de Project Management Tool.

Structure du Projet

Nous utilisons une approche Core/Shared/Features pour une meilleure séparation des préoccupations.

src/app/
├── core/ // Singletons (AuthService, Interceptors, Guards)
├── shared/ // UI Atoms (Button, Input, Card, Badge)
├── features/ // Modules métiers isolés
│ ├── dashboard/ // Widgets de métriques
│ ├── projects/ // CRUD Projets
│ └── tasks/ // Liste et détail des tâches
├── models/ // Interfaces & Type definitions
└── store/ // Gestion d'état globale

Gestion de l'état avec Signals

Depuis Angular 17, nous privilégions les Signals pour la réactivité de l'interface, notamment pour les compteurs de métriques.

Exemple d'implémentation

// dashboard.service.ts
public projectsCount = signal<number>(0);
public completedTasks = signal<number>(0);

updateMetrics(data: MetricsDTO) {
  this.projectsCount.set(data.activeCount);
  this.completedTasks.set(data.doneCount);
}

Intégration Tailwind CSS

Le Design System est injecté directement dans les classes utilitaires via le fichier de configuration.

Palette personnalisée

Utilisez les classes suivantes pour respecter la charte :

  • text-deep-navy : Titres et Sidebar.
  • bg-soft-gray : Background de page.
  • rounded-pmt : Border-radius de 12px (standard cartes).

Consommation de l'API

Chaque appel est sécurisé par un HttpInterceptor qui ajoute le token JWT.

Service Pattern

Toutes les méthodes de service retournent des Observable. Le composant s'abonne ou utilise toSignal().

// project.service.ts
getAllProjects(): Observable<Project[]> {
  return this.http.get<Project[]>(`${this.apiUrl}/projects`);
}
